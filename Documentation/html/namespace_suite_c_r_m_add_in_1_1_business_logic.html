◆ TransmissionState

enum SuiteCRMAddIn.BusinessLogic.TransmissionState
strong

States a SyncState object can be in with regard to transmission and synchronisation with CRM. See TxState.

Enumerator
NewFromOutlook 

This is a new SyncState object which has not yet been transmitted.

PresentAtStartup 

This is a SyncState representing an outlook item which was present when Outlook was started.

NewFromCRM 

This is a SyncState object representing a outlook item which has just been created from a CRM item.

Pending 

A change has been registered on this SyncState object but it has not been transmitted.

Queued 

This SyncState has been queued for transmission but has not yet been transmitted.

Transmitted 

The Outlook item associated with this SyncState has been transmitted, but no confirmation has yet been received that it has been accepted.

Synced 

The Outlook item associated with this SyncState has been transmitted and accepted by CRM.

PendingDeletion 

A state is put into state PendingDeletion if it is not found in CRM at one synchronisation run; if it is not found in the subsequent run and is still in state PendingDeletion, then it should be deleted.

Invalid 

The sync state is in an invalid state and should never be synced.
