◆ GetSenderSMTPAddress()

static string SuiteCRMAddIn.Extensions.MailItemExtensions.GetSenderSMTPAddress ( this Outlook.MailItem  olItem)
static

From this mail item, extract the SMTP sender address if any, else the empty string.

If the mail item has not yet been despatched (which in the case of a "Send and Archive" action it will not have been), the sender address will not have been filled in. Currently using the default sender address but I'm not convinced this is satisfactory.

If the sender is using Exchange (which if they're using Outlook they almost certainly are) the 'sender email address' won't be an email address, it will be a bizarre LDAP query which CRM will barf on. So resolve it if possible.

Parameters
olItemThe mail item
Returns
An SMTP address or an empty string.
