Detailed Description

Cache of CRM import/export permissions, flushed hourly.

The cache itself is global, and is held on a class (static) variable; all methods of instances of this class (actually subclass instances, since the class itself is abstract) access that single static cache.

NOTE that there is a significant difference between

module_name

and

module_key

.

module_key

is the name used internally by CRM for the module, and is unaffected by natural language changes;

module_name

will change with the natural language of the installation. Thanks to Andreas Ravnestad for this insight.
